### 1.3 页面能打开但登录失败

登录失败的根因不是小程序前端，而是后端没有真正启动成功。

你之前在根目录执行的是：

```powershell
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

这条命令在根目录执行会报：

```text
ModuleNotFoundError: No module named 'app'
```

因为后端入口在这里：

```text
backend/app/main.py
```

所以正确方式必须先进入 `backend` 目录，再启动：

```powershell
cd F:\GraduationDesign\project\DermaAgent\backend
D:\professionalSofeware\Anaconda\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

前端小程序默认请求地址是：

```text
http://127.0.0.1:8000/api/v1
```

只要后端没起来，小程序登录就一定会报：

```text
POST http://127.0.0.1:8000/api/v1/auth/login net::ERR_CONNECTION_REFUSED
```

---

### 1.4 游客模式 / 无 AppID / `webapi_getwxaasyncsecinfo:fail`

你控制台里这几类信息：

- `游客模式`
- `无 AppID 关联`
- `webapi_getwxaasyncsecinfo:fail`

本质上是微信开发者工具当前使用的是游客模式或没有绑定正式 AppID。

这类提示**不是这次登录失败的主因**，主因还是后端没起来。

但是你要注意：

1. 游客模式下，部分微信能力是受限或模拟返回的
2. 后续如果要真机调试、上传体验版、提交审核，必须使用真实小程序 AppID
3. 当前项目为了方便直接导入，构建产物里的 `project.config.json` 用的是 `touristappid`

如果你已经有真实小程序 AppID，建议把它写到：

```text
apps/mobile-user/manifest.json
```

然后重新构建一次小程序包。

## 2.1 后端一定要从 `backend` 目录启动

正确方式：

## 2.2 先确认 MySQL 和种子数据已经准备好

本项目默认数据库连接：

- Host: `127.0.0.1`
- Port: `3306`
- User: `root`
- Password: `123456`
- Database: `derma_agent`

如果数据库没导入种子数据，即使后端能启动，默认账号也可能登录失败。

SQL 执行顺序必须固定：

1. `sql/01_schema.sql`
2. `sql/02_seed_base.sql`
3. `sql/03_seed_business.sql`
4. `sql/04_seed_stats.sql`

默认测试账号：

- 用户：`user01 / 12345678`
- 医生：`doctor01 / 12345678`
- 管理员：`admin01 / 12345678`

---

## 2.3 本地联调和正式发布不是一套地址

当前小程序本地联调用的是：

```text
http://127.0.0.1:8000/api/v1
```

这只适合：

- 本地开发
- 微信开发者工具联调

如果你后面要发布，必须改成：

1. 真实 HTTPS 域名
2. 已配置到微信小程序“合法 request 域名”里的地址

否则正式版小程序会因为域名校验失败而无法请求后端。

---

## 2.4 微信开发者工具可能锁住输出目录

如果你已经打开了 `dist/build/mp-weixin`，微信开发者工具会占用这个目录。

这时即使重新构建成功，也可能出现：

- 控制台仍然显示旧报错
- 页面没有立刻刷新

处理方式：

1. 点击微信开发者工具里的“编译”
2. 如果还不对，关闭当前项目重新导入
3. 导入目录始终使用：

```text
F:\GraduationDesign\project\DermaAgent\apps\mobile-user\dist\build\mp-weixin
```

正式发布前，你一定还要补这几项：
