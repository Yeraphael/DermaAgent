## 1. Conda 环境创建命令

```powershell

conda activate derma-agent
```

- `04_seed_stats.sql`：初始化操作日志和统计快照

## 13. 用户端启动方式补充

当前用户端有两种启动方式：

- H5 本地调试：`npm run dev:user`
- 微信小程序编译调试：`npm run dev:user:mp-weixin`

H5 本地调试实际执行的 workspace 脚本为：

- `npm run dev:h5 --workspace @dermaai/mobile-user`

H5 启动脚本在执行 `vite dev` / `vite build` 前，会先检查 `apps/mobile-user/index.html` 是否存在。
如果上一次微信小程序编译中断，导致 H5 入口文件被临时隐藏，只剩下 `apps/mobile-user/.index.html.h5-backup`，
现在重新执行 `npm run dev:user` 或 `npm run build:user` 会自动恢复 `index.html`，避免访问 `http://127.0.0.1:5174/` 时出现 `HTTP ERROR 404`。
H5 构建产物会输出到 `apps/mobile-user/dist/h5`，避免和微信小程序产物目录 `apps/mobile-user/dist/build/mp-weixin` 互相冲突。

## 18. 常见报错与排查方式

### 1. `ModuleNotFoundError: fastapi`

原因：

- 当前 Conda 环境没装依赖，或者解释器不是你以为的那个

处理：

```powershell
cd F:\GraduationDesign\project\DermaAgent\backend
python -m pip install -r requirements.txt
```

### 2. MySQL 导入中文 SQL 报编码错误

原因：

- 用 PowerShell 管道直接把 UTF-8 SQL 传给 `mysql.exe` 时可能发生编码失真

处理：

- 使用 Python `subprocess` + `read_bytes()` 方式导入
- 或确保 `mysql.exe` 以 `--default-character-set=utf8mb4` 执行

### 3. `vite build` / `vite dev` 报 `spawn EPERM`

原因：

- Windows 沙箱或权限限制拦截了 `esbuild` 子进程

处理：

- 在桌面本机终端直接执行命令
- 或允许对应命令在非沙箱模式执行

### 4. uni-app 构建找不到 `sass`

处理：

```powershell
cd F:\GraduationDesign\project\DermaAgent
C:\nvm4w\nodejs\npm.cmd install --workspace @dermaai/mobile-user
```

### 5. `http://127.0.0.1:5174/` 返回 404

原因：

- 上一次执行 `npm run dev:user:mp-weixin` 或 `npm run build:user:mp-weixin` 时如果异常中断，H5 入口文件 `apps/mobile-user/index.html` 可能被临时隐藏，只剩下 `.index.html.h5-backup`

处理：

- 直接重新执行 `C:\nvm4w\nodejs\npm.cmd run dev:user`
- 或执行 `C:\nvm4w\nodejs\npm.cmd run build:user`
- 当前脚本会在启动前自动从 `.index.html.h5-backup` 恢复 `index.html`

### 6. 用户端图片上传失败

排查：

- 后端是否已启动
- `BASE_URL` 是否为 `http://127.0.0.1:8000`
- 浏览器是否允许访问本地接口

### 7. 登录提示账号或密码错误

排查：

- 确认是否已执行 4 个 SQL 文件
- 确认导入到的是 `derma_agent` 数据库
- 使用默认账号：`user01 / doctor01 / admin01`

### 8. `python -m pip install -r requirements.txt` 出现大量其它包缺依赖

现象：

- 输出里出现 `docling`、`langchain-community`、`transformers`、`rapidocr`、`openai`、`qdrant-client` 等与你当前安装命令无关的包

原因：

- 你当前的 Python / Conda 环境已经被其它项目使用过
- `pip` 在安装本项目依赖时，会顺带检查当前环境里已安装包的依赖一致性，于是把那些旧包的缺失依赖也一起报出来

处理方式：

推荐做法：

```powershell
conda create -n derma-agent python=3.13 -y
conda activate derma-agent
cd F:\GraduationDesign\project\DermaAgent\backend
python -m pip install -r requirements.txt
```

如果你必须继续使用当前旧环境：

1. 先执行 `python -m pip uninstall -y langchain-community langchain-classic docling rapidocr transformers openai qdrant-client`
2. 再执行 `python -m pip install -r requirements.txt`

补充说明：

- 这类输出里真正和本项目相关的冲突，主要是 `langchain-community` 对 `pydantic-settings` 的版本要求
- 本项目已将 `pydantic-settings` 调整为 `2.10.1`
- 如果你仍然复用旧环境，最稳妥的方式仍然是新建独立 Conda 环境

## 19. 项目完整启动顺序

1. 启动本地 MySQL 服务
2. 进入根目录，确认 `.env` 已配置完成
3. 执行 `python sql\generate_sql.py`
4. 按顺序导入 `01_schema.sql`、`02_seed_base.sql`、`03_seed_business.sql`、`04_seed_stats.sql`
5. 启动 FastAPI：进入 `backend` 目录执行 `python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload`
6. 根目录执行 `C:\nvm4w\nodejs\npm.cmd install --workspaces`
7. 启动 Web 管理端：`C:\nvm4w\nodejs\npm.cmd run dev:web`
8. 启动用户端 H5：`C:\nvm4w\nodejs\npm.cmd run dev:user`
9. 如需调试微信小程序版用户端，再额外执行：`C:\nvm4w\nodejs\npm.cmd run dev:user:mp-weixin`
10. 打开 `http://127.0.0.1:5173` 进入医生/管理员端
11. 打开 `http://127.0.0.1:5174` 进入用户端 H5
12. 使用默认测试账号完成登录、问诊提交、AI 分析查看、医生回复和后台管理联调
