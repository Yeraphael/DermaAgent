# 项目启动与部署说明

本文档对应当前产品化重构后的 `DermaAgent` 项目，适用于本地初始化、联调启动、演示部署和上线前准备。

## 1. 环境要求

- Node.js 18+
- npm 9+
- Python 3.11+
- MySQL 8+

建议：

- 为后端单独准备一个 Python 虚拟环境或 Conda 环境
- MySQL 使用 `utf8mb4` 字符集
- Windows 环境建议使用 PowerShell 或 CMD 执行命令

## 2. 安装依赖

### 前端依赖

在仓库根目录执行：

```powershell
cd F:\GraduationDesign\project\DermaAgent
npm install
```

### 后端依赖

```powershell
cd F:\GraduationDesign\project\DermaAgent\backend
python -m pip install -r requirements.txt
```

说明：

- 后端依赖以 `backend/requirements.txt` 为准
- 如果你使用的是旧 Python 环境，建议新建独立环境后再安装，避免历史依赖冲突

## 3. 环境变量配置

项目根目录提供了 `.env.example`，建议复制为 `.env` 后再修改：

```powershell
Copy-Item .env.example .env
```

关键配置如下：

```env
APP_NAME=肤联智诊 API
BASE_URL=http://127.0.0.1:8000
API_PREFIX=/api/v1
JWT_SECRET=replace-with-a-long-random-secret
JWT_EXPIRE_MINUTES=720

MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=123456
MYSQL_DATABASE=derma_agent
MYSQL_CHARSET=utf8mb4

CORS_ORIGINS=http://127.0.0.1:5173,http://localhost:5173,http://127.0.0.1:5174,http://localhost:5174
UPLOAD_DIR=backend/uploads

AI_MODE=mock
QWEN_API_KEY=
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_VISUAL_MODEL=qwen3.6-flash
QWEN_TEXT_MODEL=qwen3.6-flash
TEXT_QA_MODEL=qwen3.6-flash

VITE_API_BASE_URL=http://127.0.0.1:8000/api/v1
```

配置说明：

- `JWT_SECRET`：上线前必须替换为正式随机密钥
- `AI_MODE=mock`：默认使用本地分析兜底逻辑，便于本地启动与验收
- 若接入真实模型，请配置 `QWEN_API_KEY` 并将 `AI_MODE=real`
- `VITE_API_BASE_URL` 需与后端 API 地址一致

## 4. 数据库初始化

### 4.1 MySQL 连接建议

默认示例：

- Host：`127.0.0.1`
- Port：`3306`
- User：`root`
- Password：`123456`
- Database：`derma_agent`

### 4.2 首次建库执行顺序

首次初始化推荐按下面顺序执行：

```powershell
mysql -u root -p < sql/01_schema.sql
mysql -u root -p < sql/02_seed_base.sql
mysql -u root -p < sql/04_seed_stats.sql
```

文件说明：

- `sql/01_schema.sql`
  创建当前正式使用的核心业务表结构
- `sql/02_seed_base.sql`
  初始化普通用户、医生、管理员、患者档案、健康档案、系统配置、咨询基础数据
- `sql/04_seed_stats.sql`
  初始化控制台趋势数据、操作日志和日志页展示数据

### 4.3 可选脚本

`sql/03_seed_business.sql` 仅在你保留旧知识问答扩展表结构时按需执行。

如果当前环境没有对应旧表，请不要执行这个脚本。

### 4.4 旧库增量升级

如果你是在已有数据库基础上升级到当前版本，请先备份数据库，再执行：

```powershell
mysql -u root -p < sql/migrations/20260429_doctor_admin_productization.sql
```

该迁移会补齐：

- `consultations.abnormal_flag`
- `consultations.abnormal_note`
- `consultations.archived_flag`
- `consultations.archived_at`
- `consultation_replies.first_impression`
- `consultation_replies.care_advice`

## 9. 本地联调验证建议

建议按下面顺序验证一次完整链路：

1. 使用普通用户账号登录用户端并提交图文问诊
2. 在医生端登录已分配的医生账号，查看新问诊并提交专业回复
3. 在医生端提交 AI 结果反馈
4. 返回用户端查看医生回复是否已回写
5. 登录管理员端，在咨询记录中查看该条问诊与处理时间线
6. 在管理员端查看 AI 结果、医生反馈、异常标记与归档状态

当前版本已经完成并验证的主闭环包括：

- 用户提交问诊
- 医生端查看待处理问诊
- 医生端提交回复
- 用户端查看医生回复
- 管理端查看全平台咨询记录
- 医生反馈 AI 结果后管理端可见

## 11. 上线前准备清单

当前版本可以用于本地部署、联调演示和阶段性交付验收。

如果要直接面向真实用户上线，建议至少完成以下准备：

- 替换 `.env` 中的 `JWT_SECRET`、数据库密码和默认账号密码
- 配置正式 MySQL、备份策略与恢复演练
- 配置 HTTPS、域名、Nginx 和生产 CORS 白名单
- 把 `AI_MODE` 切换为 `real` 并配置正式模型密钥
- 根据实际环境接入对象存储或图片托管策略
- 配置日志保留、监控告警和异常通知
- 完成安全测试、压力测试和关键接口错误恢复验证

## 12. 常见问题

### 1. 后端依赖安装时报环境冲突

建议新建干净的 Python 虚拟环境后重新执行：

```powershell
cd F:\GraduationDesign\project\DermaAgent\backend
python -m pip install -r requirements.txt
```

### 2. MySQL 导入 SQL 出现中文乱码

建议使用 `utf8mb4`，并确保导入客户端使用正确字符集：

```powershell
mysql --default-character-set=utf8mb4 -u root -p < sql/01_schema.sql
```

### 3. 前端启动后接口请求失败

优先检查：

- 后端是否已启动
- `.env` 里的 `VITE_API_BASE_URL` 是否正确
- 浏览器是否能访问 `http://127.0.0.1:8000/api/v1`

### 4. 登录提示账号或密码错误

优先检查：

- 是否已执行 `sql/01_schema.sql`
- 是否已执行 `sql/02_seed_base.sql`
- 数据库是否正确导入到 `derma_agent`
- 使用的账号是否为当前文档中列出的已验证账号

### 5. 是否可以直接公网商用上线

当前版本已经具备产品化主流程和可部署能力，但仍建议先完成第 11 节中的生产加固项，再进入正式公网环境。
